PM123's Known Problems & Todo list

No. Type Sever. Topic Status Details
1 todo major Shuffle not working open There are some open issues how this should work with respect to nested playlists and the previous/next buttons when the playlist is modified.
2 fault medium PM123 sometimes crashes when starting playback testing Sometimes the decoder thread runs into a access violation at start of playback.
Reason unknown. Perhaps some threading issue.
Seems to have gone.
3 todo minor Handling of start and stop positions in nested playlists untested There are several complications when start/stop iterators of nested playlists have conflicting settings.
4 fault medium Messages about non-existing files at first start up fixed When starting the first error messages about non-existing show up. This messages should be suppressed.
5 fault medium Decoder starts at wrong priority open In playlist mode when the data for the next song is prefetched the new decoder thread does not inherit the priority from the old thread. This may cause the playback to have drop outs.
6 fault medium Deadlocks with recursive playlists open With recursive playlist structures PM123 sometimes deadlocks because the list semaphore are not requested top down. In fact there is no top and no bottom in this case. Maybe implicitly fixed by #10.
7 todo medium Invalid HTTP URLs take minutes open When an URL target is not available or invalid, it may take several minutes for PM123 to notify this. If you have many of such links e.g. in the bookmarks PM123 may be not responsive for a considerable amount of time.
This is a design problem, because any decoder plug-in might try to open the URL without success, every time waiting for a timeout.
8 fault major EAs lost when editing meta information untested The 'Edit file info' feature of mpg123 and oggplay destroys any EA information.
9 todo minor Code page handling of ID3 V1.x and ID3 V2.x tags open If the input and output code page is different the ID3 tags are not turn around stable. This causes special characters to be destroyed when saving the tag.
10 todo medium PM lock-up when slider is used while a worker thread gets information on a playlist. open Thread 1 acquires playlist mutex when slider is used. Can cause SIQ locks. Maybe the Worker should not lock the entire playlist while calculation of tech infos.
11 fault medium NavigateTo does not work if we see a slice of the current root. open If the currently visible windows root is below the currently loaded root, navigate to starts at the loaded root rather than the window's root. This consequently fails.
12 fault minor Glitches of the slider when seeking or starting at a certain location. open When playback starts at a non-zero starting point the slider jumps to zero for the time of the initialization.
