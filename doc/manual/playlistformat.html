Format of the PM123 playlist files

This format is used for PM123 playlists as well as for bookmarks, playlist manager and most recent use lists. The files are upwardly and downwardly compatible to the files of older PM123 releases, concerning the least common denominator.

Header

The head of a playlist file is a comment. This is not an essential part on input.

#
# Playlist created with PM123 1.40 alpha 3
# Do not modify!
# Lines starting with '>' are used by Playlist Manager.
#

For each song entry

  • Human readable comment with size, duration and an info string

    # 22385 kB, 21:18, 143.4kbs, 44.1kHz, Stereo

  • Alias name for display, e.g. in the bookmark list. optional

    #ALIAS Recording from Mar. 03th, 2006

  • Information on start and stop position. optional
    The unit is seconds (with fractionals). A Stop position of -1 means till the end of the source.

    #SLICE 3.921,-1.000

  • Filename or URL

    SL060319_2204.ogg

    On output PM123 writes Filenames with relative or absolute paths, and URLs. On input file:/// URLs are accepted too. In fact any URL which is accepted by the Load URL dialog is accepted. This may depend on the loaded plug-ins.

  • Technical information about the above entry

    >143,44100,0,22923026,1278.7

    This is the bitrate [kbps] or -1 if ubknown,
    the samplig rate [Hz],
    the number of channels as mode (0=stereo, 3=mono),
    the file size [bytes] or -1 if unknown and
    the songlength [s] or -1 if unknown.

For each playlist or folder entry

  • Alias name for display, e.g. in the bookmark list.

    #ALIAS Chillout-1

  • Human readable comment with file size, total duration and an info string

    # 1kiB, 1:15:18,

  • Filename or URL

    Taucher - Adult Music\Chill.lst

    On output PM123 writes Filenames with relative or absolute paths, and URLs. On input file:/// URLs are accepted too. Folder entries must end with a trailing /.

  • Technical information about the last entry

    >136,-1,-1,896,1278.7,145,19775.4,15,0

    This is the average bitrate [kbps] or -1 if unknown,
    two constants (-1,-1) indicating unknown samplingrate and channels,
    the file size [bytes] or -1 if unknown,
    the total length [s] or -1 if unknown,
    the number of song items (including subitems) in the referenced playlist,
    the total file size of all song items (including subitems),
    the number of items in the referenced playlist and
    a flag whether the content is known to contain itself recursively is added.

Footer

# End of playlist

The footer is optional on input.
