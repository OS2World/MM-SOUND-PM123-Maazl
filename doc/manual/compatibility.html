Compatibility of PM123 1.40 with earlier versions

Playlist files

PM123 1.40 stores a bunch of additional properties to playlist entries. These entries are ignored by PM123 1.32. But if you edit and save the playlist with a previous version the following properties of PM123 1.40 maybe lost:

  • Alias name
  • Start- and Stop position

Profiles and working directory

PM123 1.40 will read or convert most of the settings from earlier versions. But some information is stored in a way that earlier versions can no longer read it. They are:

  • The list of loaded plug-ins
  • The bookmark names

It is not recommended to share configuration files between different program versions.

Running two versions simultaneously

Different versions of PM123 use DLLs with the same name. Since OS/2 by default will not load a DLL when some other one with the same name is already in memory, this will usually result in a crash of the secondly started PM123 instance.

To work around this problem use SET LIBPATHSTRICT=T before you start any of the PM123 instances.

Drag and Drop

Using Drag and Drop to drag playlist items between windows of different PM123 versions results in undefined behavior. Most likely one of the applications will crash. The machine's user interface may also completely lock up, because OS/2 really dislikes crashes during Drag and Drop.
